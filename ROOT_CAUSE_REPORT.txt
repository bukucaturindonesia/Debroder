1. ELEMENT THAT WAS UNDEFINED
The React element type was the imported identifier `OrderOperationalWorkspace` used by OrderDetailAdmin. It was rendered as `<OrderOperationalWorkspace ... />`, but module resolution redirected its import to a module that did not export that name. React therefore received an undefined element type and production raised minified React error #130 with args[]=undefined.

2. EXACT SOURCE CAUSE
Before fix:
- OrderDetailAdmin imported `{ OrderOperationalWorkspace }` from `@/components/admin/OrderOperationalWorkspace`.
- Baseline tsconfig.json line 14 contained the exact path alias:
  `"@/components/admin/OrderOperationalWorkspace": ["./components/admin/CustomOrderOperationalWorkspace.tsx"]`
- The redirected file exported `CustomOrderOperationalWorkspace`, not `OrderOperationalWorkspace`.
- The real file `components/admin/OrderOperationalWorkspace.tsx` did export `OrderOperationalWorkspace`, but the exact alias prevented that file from being resolved.

After fix:
- The incorrect exact alias is removed.
- OrderDetailAdmin imports the standard and Custom implementations by their real, case-correct paths.
- `resolveAdminOrderWorkspaceKind` selects `standard` or `custom` from the immutable Custom snapshot.
- JSX uses a direct, exhaustive conditional between two statically imported functions. No dynamic component/icon map can return undefined on this path.

3. PAYMENTTRACKINGMANAGER / ORDERID CLASSIFICATION
PaymentTrackingManager did not cause or repair the undefined element type. Its deployed source obtains orderId from canonical route params through `useParams`; this contract was retained. Passing an orderId prop would not change module resolution and could not turn an undefined OrderOperationalWorkspace export into a valid React component. Classification: NOT RELATED to React #130. Only timestamp formatting inside PaymentTrackingManager was hardened defensively.

4. TIMESTAMP CLASSIFICATION
Invalid/missing timestamps are a separate defensive risk. A direct DateTimeFormat call can throw RangeError for invalid data, but that is not React #130 and does not produce an undefined React element type. All timestamp renderers in the audited Admin Order Detail dependency chain now use guarded helpers. Classification: DEFENSIVE HARDENING / SECONDARY BUG PREVENTION, not root cause.
